# **集合定义**：

集合是java中提供的一种容器，可以用来存储多个数据。

 集合类主要负责保存、盛装其他数据，因此集合类也被称为容器类。

 Java 所有的集合类都位于 java.util 包

 

# **集合和数组既然都是容器，它们有啥区别呢？**

1、数组的长度是固定的。集合的长度是可变的。

2、数组中存储的是同一类型的元素，可以存储基本数据类型值。集合存储的都是对象。

3、对象的类型可以不一致。在开发中一般当对象多的时候，使用集合进行存储。

## 双列集合Map

主要两个实现类：HashMap     类和 TreeMap 类（**util包**）

1、JAVA中**一一对应的关系**叫映射；而映射的这种对象就是用Map来存储。**键唯一，但是值可以重复。**key、value都可以为空。相同的key往往是覆盖调原来key的value的值。

2、Map接口常见的子类：

a、**HashMap**：存储数据采用的哈希表结构，元素的存取顺序不能保证一致。由于要**保证键的唯一、不重复，需要重写键的hashCode()方法、equals()方法**。

注：

**hashSet方法和hashCode**方法：

*先通过把元素得到对应的**hash**值，不同则存储；*

*相同则进行**equals**方法比较，相同元素相同，不同进行不同存储*

b、 **LinkedHashMap**：HashMap下有个子类LinkedHashMap，存储数据采用的哈希表结构+链表结构。通过链 表结构可以保证元素的存取顺序一致；通过哈希表结构可以保证的键的唯一、不重复，需要重写键的 hashCode()方法、equals()方法。

1、不同于Collection的 add() ; 而Map的put方法有返回值；

如果键值对存在就返回对应值；如果不存在就返回null，并且添加对应键值对。

eg：

```java
Object put=map.put(1,11);  

System.out.println(put);     //null

Object put2=map.put(1,11);

System.out.println(put2);     //11
```

2、keySet方法，把Map中的键key取出存储到Set集合中。

eg：

```java
Map map=new HashMap();

map.put(0,2);map.put(1,2);map.put(2,2);map.put(3,2);map.put(4,2);

Set set=map.keySet();

for(Object o:set){

System.out.print(map.get(o)+"\t");

}
```

3、entrySet方法，把集合所有键值对的对象放到Set集合。

eg：

```java
Set<Map.Entry<Integer,Integer>> set1=map.entrySet();

for(Map.Entry<Integer,Integer>entry:set1){

System.out.print(entry.getKey()+":"+entry.getValue()+"\t");

}
```

 

### **Entry**键值对对象

[ Map.Entry方法的使用](https://blog.csdn.net/weixin_44679437/article/details/106090278)

-  在Map类设计是，提供了一个（**内部）嵌套接口（static修饰的接口）**：Entry。
- Entry将键值对的对应关系封装成了对象，即键值对对象，这样我们在遍历Map集合时，就可以从每一个键值对（Entry）对象中获取对应的键与对应的值。
- Entry为什么是静态的？

​    Entry是Map接口中提供的一个静态内部嵌套接口，修饰为静态可以通过类名调用。

### Map集合遍历键值对的方式：

 

- Set<Map.Entry<K,V>> entrySet()；
- Map.Entry是Map声明的一个内部接口，此接口为泛型，定义为Entry<K,V>。

- 返回此映射中包含的映射关系的Set视图， 该方法返回值是Set集合，里面装的是Entry接口类型，即将映射关系装入Set集合。

实现步骤：

1，调用Map集合中的entrySet()方法，将集合中的映射关系对象存储到Set集合中

2，迭代Set集合

3，获取Set集合的元素，是映射关系的对象

4，通过映射关系对象的方法，getKey()和getValue(),获取键值对

 

 

# 两个接口区别：

1、Collection 中的集合，元素是孤立存在的（理解为单身），向集合中存储元素采用一个个元素的方式存储。

2、Map 中的集合，元素是成对存在的(理解为夫妻)。每个元素由键与值两部分组成，通过键可以找对所对应的 值。

3、Collection 中的集合称为单列集合， Map 中的集合称为双列集合。 需要注意的是， Map 中的集合不能包含重复的键，值可以重复；每个键只能对应一个值。
