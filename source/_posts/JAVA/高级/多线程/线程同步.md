**多线程只有在满足：多个线程共同处理一个资源才会出现线程安全问题，但是如果一个线程处理一个任务就不会有线程安全。**

#  [Java 继承Thread类和实现Runnable接口的区别](https://blog.csdn.net/xiangzhihong8/article/details/51661382?utm_medium=distribute.pc_relevant.none-task-blog-baidujs_title-1&spm=1001.2101.3001.424)

 

继承Thread类的，我们相当于拿出三件事即三个卖票10张的任务分别分给三个窗口，他们各做各的事各卖各的票各完成各的任务，因为MyThread继承Thread类，所以在new MyThread的时候在创建三个对象的同时创建了三个线程；

实现Runnable的， 相当于是拿出一个卖票10张得任务给三个人去共同完成，new MyThread相当于创建一个任务，然后实例化三个Thread，创建三个线程即安排三个窗口去执行。

 

理解：

一个是多个线程分别完成自己的任务，一个是多个线程共同完成一个任务。

 

# **synchronized**同步进程：

 

### **方法一：**同步代码块：

**synchronized(同步锁){**   **需要同步操作的代码**  **}** 

**同步锁：**

**1. 锁对象 可以是任意类型。 **

**2. 多个线程对象 要使用同一把锁**

**Object lock = new Object();**

 

代码实例： 

```java
class Ticket implements Runnable{

private int ticket=100;

private Object lock=newObject();

@Override

public void run(){

while(true){

if(ticket<=0)break;

synchronized(lock){

try{

Thread.*sleep*(100);

}catch(InterruptedException e){

e.printStackTrace();

}

String name=Thread.currentThread*().getName();

System.out.println(name+"正在卖:"+ticket--);

}

}

}

//测试：

Ticket ticket=newTicket();

//都是用一个窗口

Thread thread1=newThread(ticket,"线程1");

Thread thread2=newThread(ticket,"线程2");

Thread thread3=newThread(ticket,"线程3");

thread1.start();

thread2.start();

thread3.start();
```

理解：

Ticket 类实现了Runnable接口，实现了run（）方法，通过Thread类构造方法，让多个线程共同完成一个任务（Ticker对象的run方法），因为只new了一个对象也就只有一把锁lock。

但是如果继承Thread类new出来的对象就会各自有有自己的lock。

 

线程安全问题：synchronized 线程同步机制：只允许一个线程通过，其他线程都在等待。

但是注意：有条件判断和循环执行的时候，要注意锁的位置：

1、在包含条件判断，防止多个线程通过判断在等待锁，引发错误；

2、在循环条件下，锁要在循环内部，避免只有一个线程去完成所有循环。

### **方法三：** **Lock**锁：

 

**代码实例：**

```java
private int ticket=100;
private Lock lock=new ReentrantLock();
@Override
public void run(){
    while(true){
        lock.lock();
        if(ticket<=0){lock.unlock();break;}
        try{
            Thread.*sleep*(100);
        }catch(InterruptedExceptione){
            e.printStackTrace();
        }
        String name=Thread.*currentThread*().getName();
        System.*out*.println(name+"正在卖:"+ticket--);
        lock.unlock();

    }
}
```

**理解：**Lock接口 

**java.util.concurrent.locks.Lock** **机制提供了比synchronized代码块和synchronized方法更广泛的锁定操作, 同步代码块/同步方法具有的功能Lock都有,除此之外更强大,更体现面向对象。**

 **Lock锁也称同步锁，加锁与释放锁方法化了，**

**如下：**

| **public void lock() :**   | **加同步锁。**   |
| -------------------------- | ---------------- |
| **public void unlock() :** | **释放同步锁。** |



# **线程生命周期**

![image-20220618165306398](https://blog-images-djx.oss-cn-hangzhou.aliyuncs.com/img/202206181653461.png)

**java.lang.Thread.State**这个枚举中给出了六种线程状态：

| 线程状态                  | 导致状态发生条件                                             |
| ------------------------- | ------------------------------------------------------------ |
| New(新建)                 | 线程刚被创建，但是并未启动。还**没调用start方法**。          |
| Runnable(可运行)          | **线程可以在java虚拟机中运行的状态**，可能正在运行自己代码，也可能没有，这取决于操  作系统处理器。 Blocked(锁阻塞) 当一个线程试图获取一个对象锁，而该对象锁被其他的线程持有，则该线程进入 |
| Blocked(阻塞)；           | 当该线程持有锁时，该线程将变成Runnable状态（同步代码块/方法）。 |
| Waiting(无限等待)         | 一个线程在等待另一个线程执行一个（唤醒）动作时，该线程进入Waiting状态。进入这个  状态后是**不能自动唤醒的，必须等待另一个线程调用notify或者notifyAll方法才能够唤醒**。 |
| Timed  Waiting(计时 等待) | 同waiting状态，有几个方法有**超时参数**，调用他们将进入Timed Waiting状态。这一状态  将一直保持到超时期满或者接收到唤醒通知。带有超时参数的常用方法有**Thread.sleep 、  Object.wait**。 |
| Terminated(被终止)        | 因为run方法正常退出而死亡，或者因为没有捕获的异常终止了run方法而死亡。（**出了异常但是没有捕获，只能交给**JVM捕获，就直接终止运行） |

![image-20220618165418342](https://blog-images-djx.oss-cn-hangzhou.aliyuncs.com/img/202206181654416.png)

**六种状态转化：**

| **New**(新建) **;** | **Runnable(可运行)** **;** | **Blocked**(阻塞) | Waiting(无限 | Timed Waiting(计时等待) | Terminated(被终止) |
| ------------------- | -------------------------- | ----------------- | ------------ | ----------------------- | ------------------ |
|                     |                            |                   |              |                         |                    |

| 1、runnable --》waiting ：   | 拥有锁对象，调用wait（）方法（结束后不会保持锁对象的拥有）   |
| ---------------------------- | ------------------------------------------------------------ |
| 2、waiting--》runnable：     | 等待的线程被唤醒且获取锁对象；其他线程调用notify（）；notifyAll（）方法，获得锁对象。 |
| 3、runnable--》blocked：     | 没有锁对象或等待锁对象释放                                   |
| 4、blocked--》runnable：     | 得到锁对象                                                   |
| 5、runnable--》timed waiting | 进入记时等待，Thread.sleep（参数）方法或拥有锁对象调用 Object.wait（参数）方法 |
| 6、timed waiting--》runnable | 等待时间到了（分两种情况）；  wait（参数）：时间到且重新得到锁对象；时间未到    sleep（参数）：时间到了即可（不会释放锁对象） |
| 7、timed waiting--》blocked  | wait（参数）：时间到，但是没有得到锁对象；时间未到，也没有其他线程唤醒notify（） |
| 8、waiting--》blocked        | 线程被唤醒，但是没有得到锁对象                               |

 

 

注：

##   wait()和sleep()的区别?    

  1、wait可以指定时间也可以不指定；sleep必须指定时间。
  2、在同步中，对cpu的执行权和锁的处理不同；wait:释放执行权，释放锁。sleep:释放执行权，不释放锁。
  注：在timed waiting--》runnable 和timed waiting--》blocked   的状态变化上体现的明显。
