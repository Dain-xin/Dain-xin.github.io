什么是面向过程？

概述: 自顶而下的编程模式 把问题分解成一个一个步骤，每个步骤用函数实现，依次调用即可。

就是说，在进行面向过程编程的时候，不需要考虑那么多，上来先定义一个函数，然后 使用各种诸如 if-else、for-each 等方式进行代码执行。

最典型的用法就是实现一个简单的算法，比如实现冒泡排序。

什么是面向对象？

概述: 将事务高度抽象化的编程模式 将问题分解成一个一个步骤，对每个步骤进行相应的抽象，形成对象，通过不同对象之 间的调用，组合解决问题。

就是说，在进行面向对象进行编程的时候，要把属性、行为等封装成对象，然后基于这 些对象及对象的能力进行业务逻辑的实现。

比如：想要造一辆车，上来要先把车的各种属性定义出来，然后抽象成一个 Car 类

（1）最优化原理：假设问题的最优解所包括的子问题的解也是最优的，就称该问题具有最优子结构，即满足最优化原理。

（3）有重叠子问题：即子问题之间是不独立的，一个子问题在下一阶段决策中可能被多次使用到（该性质并非动态规划适用的必要条件，可是假设没有这条性质，动态规划算法同其它算法相比就不具备优势）。

第八天【2022-02-23】
1. Freemarker的使用，Freemarker将动态的页面（数据会动态变化的页面）进行【静态化处理】（将数据填充好了，不需要再发请求了），目的是为了减轻页面在被大量用户访问时，不断查询数据对数据库造成的压力，所以只需要针对会被大量访问时的页面进行处理就行了。
2. 静态化处理在服务提供者中进行（因为要在业务层进行方法调用），由于生成的静态页面是一种结果，而这种结果需要动态改变，所以在【执行增、删、改操作】的时候，都需要对已经生成的结果进行调整，并且由于每个静态结果页面都要有不一样的文件名，所以最优方案是用【主键id】作区分。
3. 首先需要在【WEB-INF】文件夹里面创建ftl模板文件夹（其他位置也可以，但路径要明确），然后写入【模板文件】，尾缀是【.ftl】（注意里面的变量格式），然后在实现类中写实现方法，在增、删、改操作的时候都要对模板文件进行修改。或者增、删、改的时候不进行生成操作，而是统一执行生成的操作（随便在页面加个按钮，点击时发送请求就可以了，前台和后台的套餐业务层是共用的），这样可以减轻对服务器的负担（尤其是数据量很大的时候）。
4. 完善手机验证码的发送：在用户输入手机号点击发送验证码的时候【先验证手机格式】是否正确，正确的前提下才会进行发送请求，同时进入倒计时等待，倒计时中不可以再次点击发送验证码（前端用的计时器实现，每秒执行一次）；请求发送到服务器后，服务器直接获取用户的手机号，用【第三方模板代码】将短信发送出去，但因为现在仅仅是测试使用，模板的内容和验证码都是固定的，用户手机只能收到相同的短信内容。但是生成的验证码还是随机的（这个验证码会保存到Redis数据库中，因为Redis数据库可以设置时间，时间到了这个数据就会失效了，查询也查询不到，恰好实现验证码的时效性，该保存时间可以自定义），用户需要输入这个随机的验证码才有效，而不是手机上的短信验证码。
5. 输入验证码后，在提交的时候会进行验证码的验证，【与Redis保存的数据一致】才可以通过然后进行下一步的验证，否则会提示用户验证码不正确。

Docker 容器

容器的基本概念

容器是从镜像创建的运行实例。它可以被启动、停止、删除。

Docker 仓库

仓库的基本概念

仓库是集中存放镜像文件的场所，有时候会把仓库和仓库注册服务器（Registry）看做同一事物，并不严格区分。实际上，仓库注册服务器上往往存放着多个仓库，每个仓库中又包含了多个镜像，每个镜像有不同的标签（tag）；

仓库分为公开仓库（Public）和私有仓库（Private）两种形式；

最大的公开仓库是 Docker Hub （https://hub.docker.com/），存放了数量庞大的镜像供用户下载；

用户也可以在本地网络内创建一个私有仓库；

当用户创建了自己的镜像之后就可以使用 push 命令将它上传到公有或者私有仓库，

这样下次在另外一台机器上使用这个镜像时候，只需要从仓库上 pull 下来即可；

注：Docker 仓库的概念跟 Git 类似，注册服务器也类似于 GitHub 这样的托管服务；

Docker 自定义镜像

Dockerfile 文件和基本结构

Dockerfile 用于构建 Docker 镜像，Dockerfile 文件是由一行行命令语句组成， 基于这些命令即可以构建一个镜像；

Dockerfile 分为四部分：

基础镜像信息； 维护者信息； 镜像操作指令；容器启动时执行指令；

Dockerfile 指令

1、FROM

Dockerfile 文件的第一条指令必须为 FROM 指令。

并且，如果在同一个 Dockerfile 中创建多个镜像时，可以使用多个 FROM 指令（每个镜像一次）；

2、MAINTAINER

指定维护者信息；

3、ENV

指定一个环境变量，会被后续 RUN 指令使用，并在容器运行时保持；

4、ADD

复制指定的到容器中的；

5、EXPOSE

告诉 Docker 服务端容器暴露的端口号，供互联系统使用，在启动容器时需要通过 -p 映射端口，Docker 主机会自动分配一个端口转发到指定的端口；

6、RUN

RUN 指令将在当前镜像基础上执行指定命令，并提交为新的镜像， 当命令较长时可以使用 \ 来换行；

7、CMD

指定启动容器时执行的命令，每个 Dockerfile 只能有一条 CMD 命令。如果指定了多条命令，只有最后一条会被执行。如果用户启动容器时候指定了运行的命令，则会覆盖掉 CMD 指定的命令。
