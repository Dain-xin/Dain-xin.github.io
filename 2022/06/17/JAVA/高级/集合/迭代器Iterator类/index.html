迭代器Iterator类

发表于2022-06-17|更新于2022-06-17|JAVA高级集合
|字数总计:700|阅读时长:2分钟

定义

  • Iterator迭代器：
  • 用于遍历集合所有元素，util包，只用于遍历。
  • 集合中都都有相应的迭代器方法。Iterator iterator() ；
  • 迭代：即Collection集合元素的通用获取方式。在取元素之前先要判断集合中有没有元素，如果有就把这个元素取出来，继续在判断，如果还有就再取出出来。一直把集合中的所有元素全部取 出。这种取出方式专业术语称为迭代。
  • 常用的方法：
E next() 返回迭代的下一个元素。
boolean hasNext() 如果仍有元素可以迭代，则返回 true。

对迭代器理解：

1~~当遍历集合时，首先通过调用集合的iterator()方法获得迭代器对象

然后使用**hashNext()**方法判断集合中是否存在下一个元素

如果存在，则调用**next()**方法将元素取出，否则说明已到达了集合末尾，停止遍历元素。

2~~terator迭代器对象在遍历集合时，内部采用指针的方式来跟踪集合中的元素，

在调用Iterator的**next()**方法之前，迭代器的索引位于第一个元素之前，不指向任何元素，

当第一次调用迭代器的next方法后，迭代器的索引会向后移动一位，指向第一个元素并将该元素返回，当再次调用next**方法时，迭代器的索引会指向第二个元素并将该元素返回，

依此类推，直到hasNext方法返回false，表示到达了集合的末尾，终止对元素的遍历。

但是这种迭代器的使用在JDK5 ，常用增强for循环方法（内部实现了迭代器的使用）。

所有循环体内部：会有无法取到下标出现的删除的数据错误，因为内部是类似与指针的操作但是删除数据长度变小但是指针无法控制就会导致出现错误。

增强for循环：

1、for( 元素的数据类型 变量 : Collection集合or数组 ) { //写操作代码 } 快捷键：iter

2、增强 for循环必须有被遍历的目标。

目标只能是Collection或者是数组。新式for仅仅作为遍历操 作出现。
